# Deutsch Tools

Шпаргалка и тренажёр по немецкому языку (A1 → A2).

🌐 [r4-studio.github.io/deutsch-tools](https://r4-studio.github.io/deutsch-tools)

## Файлы в репозитории

| Файл | Назначение |
|---|---|
| `index.html` | Главная страница (выбор: шпаргалка / тренажёр) |
| `cheatsheet.html` | Шпаргалка: словарь, грамматика, правила, поиск |
| `trainer.html` | Тренажёр: карточки, тесты по уровням |

| `requirements.txt` | Python-зависимости конвертера (`openpyxl`) |
| `pre-commit` | Git-хук: проверка `?v=` в `data.js` перед коммитом (копируется в `.git/hooks/`) |

## Архитектура

```
database.xlsx  ──[update_data.py]──>  data.js  ──>  cheatsheet.html
   (источник)         (конвертер)      (готовый)    + trainer.html
```

**Один источник правды — `database.xlsx`.**
Все правки контента — в xlsx, потом запуск конвертера, потом коммит.

## Как добавить слово / правило / тест

1. Открыть `database.xlsx` (Excel или LibreOffice)
2. Перейти на нужный лист (nouns, verbs, adjectives, adverbs, pronouns, numbers, phrases, sounds, terms, rules, questions)
3. Добавить строку (id можно не заполнять — сгенерируется автоматически)
4. Сохранить xlsx
5. В терминале из корня репо: `python update_data.py`

> Перед первым запуском: `pip install -r requirements.txt` (нужен только `openpyxl`)
6. `git add . && git commit -m "..." && git push`

> Изменения появляются на сайте только после пуша. Локально сайт работает оффлайн — двойной клик по `index.html`.

## Версионирование и кеш

В `cheatsheet.html` и `trainer.html` подключение `data.js` идёт с параметром версии:

```html
<script src="data.js?v=20260524"></script>
```

Этот параметр **нужно обновлять при каждом коммите** на текущую дату — иначе у юзеров останется старый кеш браузера.

Pre-commit hook (`.git/hooks/pre-commit`) автоматически проверяет дату и блокирует коммит при несовпадении — забыть невозможно.

## Как добавить запись в changelog

Открыть `changelog.js`, вставить новый объект в **начало** массива:

```js
window.CHANGELOG = [
  {
    "date": "2026-06-01",
    "title": "Название обновления:",
    "items": [
      "Что добавлено / исправлено"
    ]
  },
  ...
];
```

Дата в формате `YYYY-MM-DD`. Хранятся последние 10 записей.

## Установка и запуск (с нуля)

Если ты только что склонировал репо или сел за новый комп — вот что нужно сделать.

### 1. Что должно быть установлено

- **Python 3.10+** — для конвертера `update_data.py` (проверка: `python --version`)
- **Git** — для коммитов и пуша (проверка: `git --version`)
- **Браузер** — любой современный (Chrome, Firefox, Edge, Safari)

> Excel / LibreOffice — только если планируешь редактировать `database.xlsx`. Само приложение работает без них.

### 2. Клонировать репо

```bash
git clone https://github.com/R4-studio/deutsch-tools.git
cd deutsch-tools
```

### 3. Поставить Python-зависимости

Только если планируешь обновлять `data.js` (для просмотра/тестов — не нужно).

```bash
pip install -r requirements.txt
```

Внутри только `openpyxl` — больше ничего не требуется.

### 4. Включить pre-commit хук (опционально, но рекомендуется)

Хук блокирует коммит, если ты обновил `data.js`, но забыл бампнуть `?v=20260101` в html-файлах.

```bash
cp pre-commit .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

### 5. Открыть сайт локально

Просто двойной клик по `index.html`. Работает оффлайн, без сервера, без билда.

### 6. Типичный цикл правки

```
[правка database.xlsx]
       ↓
python update_data.py
       ↓
[проверить локально — двойной клик index.html]
       ↓
git add . && git commit -m "..."   ← хук проверит ?v= автоматом
       ↓
git push                            ← на сайте обновится через ~1 мин
```
